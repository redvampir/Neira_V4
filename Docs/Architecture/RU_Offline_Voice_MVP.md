# RU Offline Voice MVP

## Цель
Зафиксировать минимальный офлайн-стек для русского голосового контура MVP (VAD + ASR + TTS) с понятной деградацией по качеству/ресурсам и без зависимости от облака.

## Выбранный стек

| Слой | Выбор | Лицензия | Роль в MVP |
|---|---|---|---|
| VAD | **WebRTC VAD** | BSD-3-Clause | Детекция голосовой активности и отсечение тишины до ASR. |
| ASR (Primary offline) | **whisper.cpp** | MIT | Основной офлайн-распознаватель с упором на качество RU. |
| ASR (Fallback offline) | **Vosk** | Apache-2.0 | Быстрый и более лёгкий запасной ASR-путь при дефиците CPU/RAM. |
| TTS | **Piper** | MIT | Офлайн-синтез речи с лёгким рантаймом и предсказуемой интеграцией. |

## Почему выбраны именно эти зависимости

### 1) WebRTC VAD
- Де-факто стандарт для real-time VAD в CPU-сценариях.
- Хорошо подходит как pre-filter для снижения нагрузки на ASR.
- Простая стратегия интеграции: короткие фреймы аудио (обычно 10/20/30 ms) + state machine сегментации.

### 2) whisper.cpp как основной ASR
- На офлайне обычно даёт лучшее качество распознавания RU по сравнению с лёгкими классическими движками (при сопоставимых условиях).
- Гибкость по моделям/квантованию позволяет подобрать баланс «качество ↔ ресурсы» под конкретное устройство.
- Подходит для «accuracy-first» режима MVP.

### 3) Vosk как fallback ASR
- Быстрый старт, простая эксплуатация и более низкий порог по ресурсам.
- Удобен как деградационный путь: система остаётся работоспособной даже при перегрузе CPU/памяти, где whisper.cpp может не укладываться в latency budget.
- Полезен для low-end профилей и фоновых режимов.

### 4) Piper для TTS
- Полностью офлайн-контур без сетевой зависимости.
- Небольшой и практичный runtime для встраивания в desktop/edge сценарии.
- Достаточное качество для MVP-диалогов при контролируемых требованиях к естественности.

## Ограничения и риски

### CPU
- whisper.cpp существенно чувствителен к размеру модели и типу квантования: рост качества обычно стоит роста задержки.
- На слабых CPU возможны пики latency и конкуренция с игровым/интерактивным потоком.

### RAM / размер артефактов
- ASR/TTS-модели занимают заметный объём в памяти и на диске.
- Одновременная загрузка «тяжёлой» whisper-модели и TTS-голосов может выйти за целевой memory budget на low-end устройствах.

### Качество
- VAD может ошибаться на шуме/музыке (ложные срабатывания и пропуски начала фраз).
- Vosk обычно уступает whisper.cpp по точности на сложных или шумных RU-сценариях.
- Piper в некоторых голосах может звучать менее естественно, чем более тяжёлые/облачные TTS-решения.

### Лицензии и комплаенс
- Стек составлен из permissive-лицензий (BSD-3 / MIT / Apache-2.0), что упрощает коммерческое использование.
- Нужно сохранить NOTICE/атрибуции и проверить лицензионные условия конкретных моделей/голосов, поставляемых отдельно от runtime.

## Fallback-стратегия

### Базовый pipeline
`Audio In -> WebRTC VAD -> ASR (whisper.cpp primary / Vosk fallback) -> Text -> Piper TTS -> Audio Out`

### Правила переключения ASR
1. По умолчанию используется **whisper.cpp**.
2. Переключение на **Vosk** выполняется автоматически, если наблюдается одно из условий:
   - превышение целевого порога latency N раз подряд;
   - выход за порог CPU/RAM budget;
   - ошибка инициализации/инференса whisper.cpp;
   - принудительный low-power профиль устройства.
3. Возврат на whisper.cpp выполняется только после «окна здоровья» (health window), чтобы избежать флаппинга.

### Деградация качества с контролем UX
- Порядок деградации: `whisper.cpp (более точная модель) -> whisper.cpp (более лёгкая модель) -> Vosk`.
- Для пользователя: сохраняется непрерывность голосового цикла; в телеметрии фиксируется активный провайдер и причина переключения.

### Fail-safe поведение
- Если ASR временно недоступен: сохранять текстовый/кнопочный ввод как резервный канал.
- Если TTS недоступен: отдавать текстовый ответ и пробовать авто-восстановление TTS в фоне.

## Рекомендации по профилям MVP
- **Default профиль (balanced):** WebRTC VAD + whisper.cpp (small/quantized) + Piper.
- **Low-end профиль (speed-first):** WebRTC VAD + Vosk + Piper.
- **Quality профиль (accuracy-first):** WebRTC VAD + whisper.cpp (модель выше классом) + Piper, при условии соблюдения latency budget.

## Критерии готовности (DoD)
1. На целевом устройстве подтверждён непрерывный offline voice loop без сети.
2. Автопереключение `whisper.cpp -> Vosk` работает по ресурсным/ошибочным триггерам.
3. Зафиксированы и документированы пороги CPU/RAM/latency для переключения профилей.
4. В логах/метриках есть причина деградации и текущий ASR-провайдер.
5. Проверены лицензионные артефакты runtime + моделей/голосов в дистрибутиве.
