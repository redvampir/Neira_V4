# Рекомендации по C++ библиотекам для Neira (2026-03-20)

## Контекст

Документ фиксирует рекомендованные внешние инструменты для решения трёх задач:
1. Извлечение отношений из текста (учебники, справочники)
2. Инференс по семантическому графу
3. Семантическое сходство слов

**Принципиальная позиция:** все библиотеки ниже — это **инструменты предобработки и хранения**, а не замена движку рассуждений Нейры. Аналогия: словарь помогает лингвисту, но не заменяет его суждения.

---

## Задача 1 — Извлечение отношений из текста

### UDPipe ★★★ (рекомендован)

| Параметр | Значение |
|---|---|
| Язык | C++ |
| Лицензия | Apache 2.0 |
| Русская модель | Есть (Universal Dependencies) |
| Сайт | https://ufal.mff.cuni.cz/udpipe |
| Размер модели | ~50 МБ |

**Что даёт:** dependency parse предложения → структурированные тройки `(подлежащее, сказуемое, дополнение)` → прямой вход для `FStructuredClaim`.

**Пример:**
```
"Кот является млекопитающим"
→ кот [nsubj] → является [ROOT] → млекопитающим [nmod]
→ FStructuredClaim { Subject="кот", Predicate="является", Object="млекопитающим", RelationType=Hypernym }
```

**Место интеграции:** Python-скрипт обогащения базы (`build_semantic_graph.py`) — не в рантайм Нейры.

**Важно:** UDPipe — структурный анализатор, не статистический предсказатель смысла. Он даёт синтаксическую структуру; интерпретацию структуры выполняет Нейра по своим правилам.

| Параметр | Значение |
|---|---|
| Язык | Python |
| Лицензия | MIT |

## Задача 2 — Инференс (цепочки выводов)

### SQLite Recursive CTEs ★★★ (рекомендован, уже есть)

**Новых зависимостей не требует.** SQLite уже используется в `FSemanticGraph`. Recursive CTEs дают транзитивное замыкание бесплатно:

```sql
-- Все гиперонимы слова (глубина до 10)
WITH RECURSIVE hyper_chain(lemma, ancestor, depth) AS (
  SELECT w.lemma, r.target_lemma, 1
  FROM relations r JOIN words w ON r.word_id = w.id
  WHERE r.relation_type = 'hypernym'
  UNION ALL
  SELECT h.lemma, r.target_lemma, h.depth + 1
  FROM hyper_chain h
  JOIN relations r
    ON r.word_id = (SELECT id FROM words WHERE lemma = h.ancestor)
  WHERE r.relation_type = 'hypernym' AND h.depth < 10
)
SELECT * FROM hyper_chain WHERE lemma = 'кот';
-- кот → животное → живое_существо → объект → ...
```

**Место интеграции:** `FSemanticGraph` — добавить метод `GetTransitiveRelations(FString Lemma, ERelationType Type, int32 MaxDepth)`.

**Контракт метода:**
- Вход: лемма, тип отношения, максимальная глубина (1–15)
- Выход: `TArray<FSemanticRelation>` с полем `Depth`
- Инвариант: никогда не возвращает циклы (SQLite recursive CTE с depth guard)

## Задача 3 — Семантическое сходство слов

### fastText ★★ (рекомендован для фазы 2)

| Параметр | Значение |
|---|---|
| Язык | C++ |
| Лицензия | MIT |
| Источник | Facebook Research |
| Размер векторов (рус.) | 1–7 ГБ (зависит от модели) |

**Что даёт:** косинусное сходство слов → автоматическое обнаружение `Collocation`/`SemanticNeighbor` связей даже для слов, отсутствующих в графе явно.

```cpp
// "синтаксис" ↔ "грамматика" = 0.87 → добавить Collocation связь
// "кот" ↔ "собака" = 0.73 → возможный SemanticNeighbor
```

**Важно:** векторы — детерминированная таблица поиска, не вероятностная модель рассуждений. Результат одинаков при каждом вызове.

**Место интеграции:** Python-скрипт обогащения или offline pre-processing, не в рантайм.

## Инвариант архитектуры (критически важно)

> Все перечисленные библиотеки — **инструменты получения структурированных данных**, а не движок рассуждений.
>
> Движок рассуждений Нейры — это `FBeliefEngine` + `FHypothesisStore` + `FDialoguePipeline`.
> Они работают с уже структурированными входами. Библиотека может дать тройку `(кот, является, животное)`,
> но решение о том, подтверждать ли эту гипотезу, с каким весом, по какому источнику —
> принимает исключительно Нейра.

Нарушение этого инварианта = превращение Нейры в систему с непрозрачными решениями.
