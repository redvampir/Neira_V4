# ADR 0002: Классификация архитектуры Нейры как Symbolic Belief Agent (SBA)

- Статус: accepted
- Дата: 2026-03-20
- Владелец: Team
- Заменяет: -
- Заменён: -

## Контекст

Проект Neira реализует языкового агента принципиально отличного от LLM. Без чёткой
классификационной метки возникает риск "дрейфа архитектуры" — постепенного добавления
статистических компонентов, которые размывают гарантии объяснимости и верифицируемости.

Кроме того, при онбординге новых участников и в документации требуется краткий термин,
точно описывающий тип системы.

## Решение

Нейра классифицируется как **Symbolic Belief Agent (SBA)** — символический агент с
управляемыми убеждениями.

Ближайший академический прототип — **BDI-агент** (Belief-Desire-Intention):

| BDI-концепт | Реализация в Нейре |
|---|---|
| **Beliefs** — убеждения о мире | `FHypothesisStore`: три состояния Pending → Confirmed → Verified |
| **Desires** — цели | Цель текущего диалога / сессии |
| **Intentions** — намерения | `FActionRequest` → `FDialoguePipeline` |

### Что термин SBA запрещает

Следующие подходы **нарушают** инвариант SBA и требуют отдельного ADR для обоснования:

- Встраивание локальной LLM в рантайм Нейры (даже малой — 1B, 3B параметров)
- Вероятностный выбор ответа без явного семантического основания
- Любой компонент, решение которого нельзя воспроизвести детерминировано по входным данным

### Что термин SBA разрешает

- Детерминированные инструменты предобработки (UDPipe, fastText как таблица поиска)
- SQLite для хранения и вывода по правилам
- Правило-ориентированный NLG (см. ADR-0003)
- Внешние LLM **только** как источник для обогащения базы знаний (тег `ExternalLLM`,
  вес источника по ADR-0001), но не в рантайм диалога

## Альтернативы

1. **Expert System** — исторически точно, но несёт негативные коннотации "системы 80-х",
   не отражает динамическое управление убеждениями.
2. **Rule-Based Chatbot** — принижает архитектуру, не отражает инференс и гипотезный механизм.
3. **Neuro-Symbolic AI** — некорректно: нейронных компонентов в рантайме нет.
4. **Knowledge-Based System (KBS)** — точно, но слишком широко, не акцентирует диалоговую
   природу и механизм убеждений.

## Последствия

- Плюсы: чёткая архитектурная граница; новые компоненты легко проверить на соответствие;
  онбординг ускоряется — один термин заменяет длинное объяснение.
- Минусы/риски: термин SBA не является широко принятым стандартом — при общении с внешними
  специалистами потребуется краткое пояснение через BDI.

## Rollback / миграция

ADR не влияет на код — только на терминологию и ревью-процесс. Откат: принять новый ADR
с уточнённой классификацией.
