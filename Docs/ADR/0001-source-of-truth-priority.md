# ADR 0001: Приоритет источников истины при конфликте интерпретаций

- Статус: accepted
- Дата: 2026-03-19
- Владелец: Team
- Заменяет: -
- Заменён: -

## Контекст

В проекте есть несколько источников: словарь, подтверждения пользователя, ручные правки команды.
Без явного приоритета возможны конфликтующие трактовки и непредсказуемое поведение.

## Решение

1. Финальный приоритет фиксируется командой через ADR-процесс.
2. До принятия нового ADR действует последняя стабильная матрица приоритетов.
3. Любое изменение приоритета сопровождается датой, владельцем, причиной и планом валидации.

## Альтернативы

1. Жёстко зашить приоритет в коде без процесса пересмотра.
2. Разрешать приоритет локально в каждом модуле без общей политики.

## Последствия

- Плюсы: управляемость, трассируемость, предсказуемость поведения.
- Минусы/риски: процесс требует дисциплины и дополнительной документации.

## Rollback / миграция

Если новая матрица приоритетов ухудшает качество, откатываемся к последнему accepted ADR и повторно прогоняем regression-наборы.
